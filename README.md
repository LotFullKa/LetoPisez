# Telegram D&D Chronicle Bot (Цифровой Летописец)

Этот проект представляет собой Telegram-бота на Python, который принимает текстовые и голосовые логи D&D-сессий, обрабатывает их через Google Gemini и автоматически поддерживает Obsidian Vault с Dataview-индексами и Git-синхронизацией.

## Основные возможности

- Приём текстовых и голосовых сообщений с логами сессий.
- Транскрибация аудио через Gemini.
- Извлечение сущностей (NPC, Locations, Quests, Items) и создание/обновление соответствующих заметок в Obsidian Vault.
- Автоматическое добавление YAML-frontmatter для Dataview.
- Поддержание Dataview-индексов (`_Index.md`) в ключевых папках.
- Автоматический `git add`, `git commit`, `git push` после изменений.

## Стек

- Python 3.11+
- `python-telegram-bot`
- `google-generativeai` (Gemini)
- Obsidian Vault в локальной файловой системе (с Git-репозиторием поверх).

## Структура проекта

- `bot/` — Telegram-бот и обработчики.
- `services/` — интеграции с Gemini, Git, менеджер Vault, модели сущностей.
- `config/` — конфигурация и загрузка настроек.
- `vault/` — пример корня Obsidian Vault (можно перенастроить через конфиг).

## Быстрый старт

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Создайте файл окружения на основе примера:

```bash
cp .env.example .env
```

3. Заполните `.env`:

- Telegram Bot Token
- Список разрешённых Telegram ID
- Путь к вашему Obsidian Vault
- Ключ API для Gemini
- Настройки Git (опционально)

4. Запустите бота локально:

```bash
python -m bot.main
```

## Запуск через Docker

1. Соберите образ (опционально, `docker-compose` сделает это сам):

```bash
docker build -t dnd-chronicle-bot .
```

2. Убедитесь, что рядом с проектом есть папка `vault/`, которая является (или смонтирована как) вашим Obsidian Vault с Git-репозиторием.

3. Заполните `.env` (как описано выше). Для Docker по умолчанию `VAULT_PATH=/vault`, а папка `vault/` из хоста монтируется в `/vault` внутри контейнера.

4. Запустите через `docker-compose`:

```bash
docker-compose up -d
```

Бот поднимется в контейнере, будет писать изменения в смонтированный Vault и выполнять `git add/commit/push` внутри этого же каталога.

## Безопасность

- Доступ к боту ограничен списком разрешённых Telegram ID.
- Все секреты хранятся только в переменных окружения (`.env` не добавляется в Git).
- Запись Markdown-файлов реализуется атомарно (через временный файл и `os.replace`).
